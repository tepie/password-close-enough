password-close-enough by tepietrondi

Checks to see if a password is close enough to an actual password for a user name.

Password close enough allows implementation of password validation to be "close enough". If you are prone to typing your password incorrectly sometimes and wish sites would just accept your mistypes, then this is the implementation for you. Please note, if anyone actually implements this on their site, you must be stupid. This is for fun and a concept. This should never be allowed unless you would like to give the user more fine grained feedback on how wrong their password is.

Dependencies

jQuery, Hatred for security, 4 node Hadoop cluster, App engine, EC2 instance, download the 30 gig VMWare image... just kidding, you need jQuery only.

Install

To run the demo, checkout and run an HTTP server in the directory where you checked it out. I use: 

python -m SimpleHTTPServer


To add to your site or project, follow the demo in the index.html. You need to pass in the original and attempted password. You also need a DOM object for status messages.
